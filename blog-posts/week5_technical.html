Method to my Madness

Methods are the muscles of our ruby classes. Much like our muscles help our body to do the things we want, methods help our programs to do the things that we want. Just like we have different types of muscles (fast-twitch vs slow-twitch), there are also two different types of methods that have different requirements and different uses.

Instance methods, as their name implies, are methods that are run on an instance of a class. The majority of methods you make will most likely be instance methods so that you can manipulate objects that you create. To run an instance method, you must first create an object first, then run the method on that object:

class Example
def show #instance variable
puts "This is an instance example!"
end
end
learn = Example.new
learn.show => "This is an instance example"

Class methods, don't require you to create an object first and can just be ran on the class. When creating class methods, simply put 'self.' in front of the method name and you will have a class method:

class Example
def self.show
puts "This is a class method example"
end
end

Example.show => "This is a class method example"

As stated before, the majority of the time you are going to use instance methods to manipulate data. The reason is because many times you will want to perform more than one function on an object, and you can't do that just using class methods. I know it may seem like using class methods is a little bit easier going of my extremely simple example, but it most real-world applications, instance methods are the way to go unless you have an entire class dedicated to doing blanket function.
